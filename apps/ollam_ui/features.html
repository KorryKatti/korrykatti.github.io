System Documentation

Technical specifications and capabilities for the Ollama AI UI.

1. Python Code Interpreter

The system provides an isolated Python 3.x execution environment. Code is run within a restricted Alpine Linux sandbox to provide computational capabilities to the AI model.

  • Runtime: Python 3.11 (Alpine Linux).
  • Performance: Packages are pre-cached and managed by the uv high-speed package manager.
  • Isolation: Subprocess execution under a non-root runner user with env scrubbing.
  • Security: All execution requests require a Proof-of-Work (PoW) challenge solution (SHA-256) to prevent spam and automated abuse.
  • Timeout: Hard process kill-switch active after 60 seconds.

2. Data Visualization (Graphing)

The system interprets numerical data and converts it into visual artifacts using Chart.js. The AI generates the required schema, which is then rendered on the client side.

  • Supported Charts: Line, Bar, Pie, Radar.
  • Rendering: HTML5 Canvas via Chart.min.js.
  • Interactivity: Supports dynamic tooltips and real-time data inspection.

3. Web Search & Intelligence

Real-time internet access is facilitated through a Python-based search proxy, bridging the model to DuckDuckGo and other external providers.

  • Search Logic: AI-driven query refinement to improve result relevancy.
  • Intel Retrieval: Pulls headlines, body snippets, and source URLs.
  • Image Generation: Integrated Puter.js support for generating visuals via Imagen 4.

4. Document & Vision Processing

Supports local processing of uploaded assets for privacy-first data extraction.

  • OCR Engine: Tesseract.js running locally in the browser worker.
  • PDF Logic: PDF.js parsing for extracting text contexts from document attachments.
  • Compute Location: 100% Client-side processing (data does not leave the browser for extraction).

5. Advanced Reasoning

Enables multi-turn self-critique cycles ("Deep Think") to handle high-complexity logic and error correction.

  • Iteration: Configurable loops (1-5) for internal critique before the final response.
  • Error Handling: Automated Python code repair logic on execution failure.
